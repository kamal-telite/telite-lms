3
Final Hybrid PostgreSQL
Backend → TELITE_POSTGRES_* → telite_backend DB ✓
Moodle → MOODLE_DB_* → moodle DB ✓
Postgres container → init.sql → 2 DBs + 2 users ✓
Bottom line: The backend code is already capable of full separation. The remaining work is infrastructure: (1) Create a Postgres init SQL script that bootstraps both moodle and telite_backend databases with separate users. (2) Rewire docker-compose.yml so the backend container reads TELITE_POSTGRES_* instead of ${POSTGRES_*}. (3) Update .env.example and docs to reflect the clean two-database model. SQLite stays as the simplest local-dev fallback — no need to remove it.
